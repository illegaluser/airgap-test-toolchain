# 결제 모듈 v1.2 — 기능 명세

## 1. 개요

본 모듈은 사용자의 결제 요청을 처리하고, 외부 PG (Payment Gateway) 와 통신하여
거래를 완료한다. 운영 환경에서는 PG 와의 연동이 필수이며, 테스트 환경에서는
mock PG 가 동작한다.

## 2. 주요 기능

### 2.1 카드 결제

- 사용자가 결제 페이지 (`/payment`) 에 접근
- 카드 번호 / 만료일 / CVV 입력 후 결제 버튼 클릭
- 시스템이 외부 PG 에 거래 요청 전송 (`POST /pg/charge`)
- PG 응답에 따라 성공 / 실패 화면 표시

### 2.2 결제 취소

- 결제 완료 후 24시간 이내 취소 가능
- 사용자는 결제 내역 페이지 (`/payments/{id}`) 에서 취소 버튼 클릭
- 시스템이 PG 에 취소 요청 전송 (`POST /pg/refund`)

## 3. 입력 검증

| 필드 | 형식 | 제약 |
| --- | --- | --- |
| 카드 번호 | 16자리 숫자 (`-` 또는 공백 허용) | Luhn 알고리즘 통과 |
| 만료일 | MM/YY | 현재 시점 이후 |
| CVV | 3-4자리 숫자 | 카드 종류별 길이 검증 |
| 금액 | 양의 정수 | 1 ~ 9,999,999 (최대 KRW 999만) |

## 4. 화면 요소 (id 기준)

| id | 타입 | 설명 |
| --- | --- | --- |
| `#card-number` | input | 카드 번호 입력 |
| `#card-expiry` | input | 만료일 입력 (MM/YY 패턴 mask) |
| `#card-cvv` | input | CVV 입력 (length 3-4) |
| `#amount` | input | 결제 금액 |
| `#pay-btn` | button | 결제 버튼 (텍스트 "결제하기") |
| `#cancel-btn` | button | 결제 취소 (결과 페이지에서) |
| `#status-text` | span | 결과 상태 ("성공" / "실패: <사유>") |

## 5. 에러 처리

- PG 통신 실패 (timeout / 5xx): 사용자에게 "잠시 후 다시 시도" alert.
  자동 재시도 없음 (사용자 trigger).
- 카드 정보 오류 (4xx from PG): "카드 정보를 확인하세요" alert.
- 잔액 부족: "잔액이 부족합니다" alert.

## 6. 보안 / 감사

- 카드 번호는 마스킹 (`****-****-****-1234`) 후 로그에 기록.
- 거래 ID 는 UUID v4. 모든 요청 / 응답 로그에 포함.
- PCI-DSS 부분 준수 (PG 위임으로 카드 데이터 비저장).
