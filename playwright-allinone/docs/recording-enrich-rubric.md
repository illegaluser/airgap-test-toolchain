# TR.5 — Recording → Doc 역추정 평가 rubric (R-Plus DoD 검증용)

PLAN §"TR.5" 의 평가 rubric 5점. 사람 평가자 2인이 독립 채점 후 평균.

## 채점 기준

| 점수 | 기준 |
| --- | --- |
| 5 | 모든 6 섹션(목적/범위/사전조건/단계/예상결과/검증기준) 완비 + 단계 누락 0 + 환각 0 |
| 4 | 6 섹션 완비 + 단계 누락 1 이내 + 환각 0 |
| 3 | 1개 섹션 누락 또는 단계 누락 2 + 환각 0 |
| 2 | 2개 섹션 누락 또는 환각 1건 |
| 1 | 3개 이상 섹션 누락 또는 환각 2건+ |

## 용어 정의

- **6 섹션**: `## 목적`, `## 범위`, `## 사전조건`, `## 단계`, `## 예상 결과`, `## 검증 기준`. h2 헤딩 일치 (대소문자·공백 무관).
- **단계 누락**: 입력 `scenario.json` 의 step 중 출력 `## 단계` 섹션에 1:1 로 매핑되지 않은 step 의 수.
  - navigate / click / fill / press / select / check / hover / drag / upload / scroll / wait / verify / mock_status / mock_data 모두 포함
  - description 단어가 정확히 일치할 필요는 없음. 의미적으로 같은 행위를 가리키면 1:1 인정.
- **환각**: 입력 시퀀스에 없는 UI 요소·액션·페이지·자격증명을 새로 만들어내는 경우.
  - 예: 입력에 `email` 필드만 있는데 `phone` 필드 검증 단계 추가 → 환각 1건
  - 예: 입력에 없는 `2FA` 단계 추가 → 환각 1건
  - 합리적 일반화 (`button[type=submit]` → "제출 버튼") 는 환각 아님.

## 채점 절차

1. 평가자 2인이 독립 채점.
2. 평균이 R-Plus DoD 통과 기준 (4.0+).
3. 두 평가자 점수 차가 2점 이상이면 토론 후 재채점.
4. 5건 모두 채점한 평균이 4.0+ 이면 R-Plus 시나리오 A DoD 통과.

## 자동 사전 검사 (선택)

본 rubric 의 사람 평가 전, 다음 자동 사전 필터 적용 가능:

```python
# 6 섹션 모두 존재하는지
required = ["목적", "범위", "사전조건", "단계", "예상 결과", "검증 기준"]
missing = [s for s in required if f"## {s}" not in markdown]
# 사전 필터: missing > 2 면 사람 평가 없이 1점 처리
```

`enricher.py` 가 응답 후 자동으로 누락 섹션을 로깅한다 (`[enricher] 응답에서 누락된 섹션: [...]`). 운영자가 doc_enriched.md 를 검토할 때 이 로그를 함께 참조.
